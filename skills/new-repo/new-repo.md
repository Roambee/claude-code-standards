# /new-repo — Architecture Doc Generator

Run once per repo after cloning. Generates `architecture.md` (index) + the `docs/architecture/` tree + scoped `CLAUDE.md` files per package. Required before Claude will write or edit any file in the repo (Hook 1 enforces the root `architecture.md`).

**Announce at start:** "Running /new-repo to generate the architecture doc tree for this repository."

---

## Step 1: Gather Context

```bash
cat package.json 2>/dev/null || cat pyproject.toml 2>/dev/null
cat README.md 2>/dev/null | head -80
ls packages/ apps/ src/ 2>/dev/null
cat .env.example 2>/dev/null
git log --oneline -10
git remote get-url origin
```

Also detect package types:
```bash
# For each directory in packages/ or apps/:
for pkg in packages/* apps/* 2>/dev/null; do
  [ -d "$pkg" ] || continue
  if [ -f "$pkg/pyproject.toml" ]; then echo "$pkg: python-ai"
  elif grep -q '"@nestjs/core"' "$pkg/package.json" 2>/dev/null; then echo "$pkg: nestjs-api"
  elif grep -q '"react"' "$pkg/package.json" 2>/dev/null; then echo "$pkg: react-mfe"
  else echo "$pkg: unknown"
  fi
done
```

---

## Step 2: Create the Directory Tree

```bash
mkdir -p docs/architecture/decisions
for pkg in packages/* apps/* 2>/dev/null; do
  [ -d "$pkg" ] || continue
  MODULE=$(basename "$pkg")
  mkdir -p "docs/architecture/$MODULE"
done
```

---

## Step 5: Generate Scoped `CLAUDE.md` Files

Create a `CLAUDE.md` inside each package directory matched to its type. Claude loads the nearest `CLAUDE.md` automatically — this scopes context so frontend devs don't load AI standards and vice versa.

**NestJS API packages:**
```markdown
# [Package Name] — Claude Context

## Standards
- Use `ResponseHandlerService` for all responses — never `res.json()`
- All endpoints need `@ApiOperation` + `@ApiResponse` Swagger decorators
- Auth is handled by CAS middleware — do not implement auth logic here
- Use `Logger` from `@nestjs/common` — never `console.log`

## Architecture
See [../../docs/architecture/<module>/overview.md](../../docs/architecture/<module>/overview.md) before making structural changes to this package.
```

**React microfrontend packages:**
```markdown
# [Package Name] — Claude Context

## Standards
- User-facing errors: `EventEmitter.emit('showSnackbar', { message, severity })` — never `console.error`
- No imports from other MFE packages except `@decklar/client-utility` and `@decklar/shared`
- Server state: React Query. UI state: local `useState` only.

## Architecture
See [../../docs/architecture/<module>/overview.md](../../docs/architecture/<module>/overview.md) before making structural changes to this package.
```

**Python AI packages:**
```markdown
# [Package Name] — Claude Context

## Standards
- All LLM calls via OpenRouter through `LLMProvider` — no direct Anthropic/OpenAI SDK
- `async/await` for all LLM calls — never blocking HTTP
- Prompts live in `prompts/<feature>.md` — never inline strings > 100 chars
- Log every LLM call: model, tokens_in, tokens_out, latency_ms, request_id

## Architecture
See [../../docs/architecture/<module>/overview.md](../../docs/architecture/<module>/overview.md) before making structural changes to this package.
```

---

## Step 6: Commit Everything

```bash
git add architecture.md docs/architecture/ packages/*/CLAUDE.md apps/*/CLAUDE.md
git commit -m "docs: add architecture index, module deep files, and scoped CLAUDE.md files"
```

Output: "✅ Architecture tree created. Hook 1 will now pass. Hook 16 will surface the relevant module doc when editing any package."

---

## Step 7: Next Step

Tell the developer: "Run `/architecture` any time you need to add a module or record an architecture decision. The tree is set up — Claude will load only the context relevant to the package you're editing."

## Opt-out

If this repository should be exempt from the architecture.md requirement (POC, throwaway script):

```bash
touch .no-architecture-check
git add .no-architecture-check
git commit -m "chore: opt out of architecture check (POC repo)"
```
